# 小程序帧规范 v1.0（微信小程序）

> **适用对象**：Claude PM Agent
> **读取时机**：产品涉及微信小程序时传入
> **前置依赖**：必须与 `proto_contract.md` 同时传入；移动端操作布局规则和弹框规则见 `proto_cross_platform.md`

---

## 一、标准画布尺寸

| 要素 | 规格 |
|------|------|
| 画布宽度 | **375px** |
| 顶部宿主区 | 预留 44px 占位（胶囊按钮区 + 状态栏），使用灰色占位块标注「宿主导航区-不可干预」，不绘制任何自定义内容 |
| 自定义导航区 | 位于宿主区下方，可自定义标题和左侧返回操作，高度约 44px；次级页面用 pub `fb-navbar` + `fb-nav-back`（见 `bujue-design-system/fb-fallback-manifest.md §3.12`；S4-35 校验返回入口存在）|
| 底部 Tab 栏 | 若宿主接管 Tab：页面内不绘制底部 Tab；若小程序自有 Tab：高度 50px，与 APP 端底部 Tab 规范一致 |
| 底部安全区 | iOS 预留 34px |

> 小程序运行于微信等宿主 App 容器内，Chrome 结构由宿主控制，**PM 不得在原型中自行绘制宿主组件**。

---

## 二、`[Must]` 与 APP 端关键差异

以下差异须在触点卡片「边缘情况」列逐一标注：

| 差异点 | 处理方式 |
|--------|---------|
| 原生组件（picker、map、camera 等）外观由宿主决定 | 原型中用占位块 + 文字说明替代，不绘制自定义外观 |
| 需宿主授权的能力（摄像头、位置、文件读写等） | 在触点「边缘情况」列注明「弹出宿主授权弹窗，用户拒绝后的降级路径」 |
| 微信小程序页面栈最多 10 层 | 在流程图中标注返回策略，不得出现无法返回的路径 |
| 分享、保存图片依赖宿主菜单 | 触点行为描述写「呼起系统分享菜单」，不绘制自定义分享界面 |

---

## 三、弹框与浮层

小程序内弹框/浮框规则与 APP 端保持一致，参见 `proto_cross_platform.md` §弹框决策树与规范。

原生组件（如 `<picker>`）产生的系统弹框不在本规范管理范围内，原型中用占位说明替代。
