## 二、手机帧规范

### 帧高度模式

根据页面内容量，选择以下两种高度模式之一：

**固定标准高度（默认）**：内容可在标准高度内完整显示时，使用 `aspect-ratio: 440 / 956` 固定宽高比。内容不足时空白区域保持页面背景色。

**自适应高度（长页面模式）**：内容量超出标准高度时，帧高度自适应内容（`height: auto`，最小高度不低于标准高度）。实现方式：在页面帧元素上附加 CSS class `phone-screen-long`。

```css
.phone-screen-long {
  aspect-ratio: auto;
  height: auto;
  min-height: /* 标准高度 */;
}
```

**判断依据**：
- 列表页、空态、加载态、短表单（≤ 4 个字段）→ 固定高度
- 多分区设置页、长表单（≥ 5 个字段）、多段落详情页 → 自适应高度
- 底部操作栏使用 `position:absolute;bottom:0` 的页面 → 保持固定高度，维持吸底栏定位正确性

### 手机帧布局结构

```
Dynamic Island 占位（59pt）
导航栏（44pt）
内容区（可滚动）
底部操作栏（按需）
底部安全区（34pt，iOS）
```

### 手机横屏适配

手机端原型默认为竖屏。以下组件须额外绘制横屏帧：

| 需横屏适配的组件/页面 | 说明 |
|-------------------|------|
| 图片预览 / 图片浏览器 | 横屏最大化展示图片，支持双指缩放 |
| 视频播放器 | 横屏全屏播放，显示播放控制栏 |
| PDF 预览 | 横屏展示更宽内容，可切换单页/双页 |
| 相机/扫码 | 系统级横竖屏随陀螺仪旋转 |

手机横屏帧画布尺寸：**956 × 440 pt**，状态栏降至 24pt，动态岛位于屏幕左侧。

## 三、PAD 帧规范

> **[Must] SSOT 真源**：本章是 **PAD 布局规则的单一权威源**——含默认方向、布局结构、简化规则（同手机帧豁免条件）、骨架屏标准、末位固定操作栏 5 类规则。`proto_cross_platform.md §六`「端口差异说明」+ `prd_expression_standard.md` 段「frame-platform-note」均为本章派生表达（前者业务示例、后者 HTML 渲染约束），**改动须先改本章真源、再同步派生**；PM 写 spec 引用 PAD 规则时**只引本章**，避免多源抄引致 spec 内部字面冲突。详见 `ssot_anchors.md #47`。

### `[Must]` PAD 默认方向

PAD 帧**默认展示横屏态**，画布尺寸 **1376 × 1032 pt**（宽 × 高）。若横竖屏布局存在差异，需额外绘制竖屏帧（1032 × 1376 pt）。

### PAD 布局结构

```
手机帧（440 × 956 pt）：
  顶部状态栏 + Dynamic Island（59pt）+ 导航栏（44pt）+ 内容区 + 底部安全区

PAD 帧（1032 × 1376 pt 竖屏 / 1376 × 1032 pt 横屏）：
  顶部状态栏（24pt）+ 导航栏（50pt）+ 内容区（可利用宽屏做双栏或侧边栏）+ 底部安全区（20pt）
```

### PAD 简化规则

APP 端原型须同时包含手机帧和 PAD 帧，两者共用同一套状态枚举表。

- PAD 帧与手机帧布局**完全一致**（仅尺寸不同）→ 允许绘制骨架屏帧，标注「布局同手机帧，尺寸适配 PAD」
- PAD 帧与手机帧有**任何操作路径差异** → 必须绘制完整 PAD 帧

### `[Should]` PAD 骨架屏标准

PAD 骨架屏必须是具有真实视觉结构的帧，须包含灰色矩形占位块（`var(--fb-bg-5)`）、分区色块差异或分隔线。**纯文字标注卡不等同于骨架屏，不可替代视觉帧。**

| 要求项 | 规则 |
|-------|------|
| 功能分区 | 导航区、内容区、底部操作区须通过背景色差异或分隔线明确区分 |
| 按钮占位 | 底部操作区**一个按钮对应一个独立占位块**，数量必须与对应手机帧一致 |
| 关键入口 | 须保留核心操作按钮的占位（如「提交」「确认」等主操作） |

### `[Should]` PAD 横竖屏适配

| 情况 | 处理方式 |
|------|---------|
| 横竖屏布局完全一致（仅宽度变化） | 标注「横竖屏布局一致，宽度自适应」，无需单独绘制竖屏帧 |
| 横竖屏布局存在差异 | 在 PAD 帧区块内额外绘制竖屏帧（1032 × 1376 pt），标注「竖屏态」 |

**常见需绘制竖屏帧的场景**：横屏双列 → 竖屏单列；横屏固定侧边栏 → 竖屏汉堡菜单；横屏左右分栏 → 竖屏上下堆叠。

### PAD 多品牌设备适配

PAD 端原型以 iPad Pro M5 13英寸为基准，同时兼容主流 Android PAD：

| 品牌/系列 | 代表型号 | 屏幕比例 | 窗口化支持 |
|---------|---------|---------|----------|
| Apple iPad Pro | M5 13英寸 | 4:3 | Stage Manager |
| Apple iPad Air | M3 13英寸 | 4:3 | Stage Manager |
| Samsung Galaxy Tab | S10 Ultra | 约 16:10.3 | DeX 桌面模式 |
| Huawei MatePad | Pro 13.2 | 约 16:11 | 悬浮窗口 |
| Xiaomi Pad | 7 Pro | 约 3:2 | 分屏/小窗 |
| 通用 Android PAD | — | 16:10 ~ 16:9 | 分屏/小窗 |

其余设备差异通过响应式断点规则覆盖，不单独绘制其他品牌原型帧。

### PAD 响应式布局断点

| 宽度范围 | 布局模式 | 说明 |
|---------|---------|------|
| < 440 pt/dp | 手机布局 | 完全切换为单列手机布局 |
| 440 ~ 700 pt/dp | PAD 紧凑布局 | 单列内容区，左侧边栏收起或以汉堡菜单替代 |
| > 700 pt/dp | PAD 宽屏布局 | 双列或固定左侧边栏布局 |

**最小窗口约束**：宽度 ≥ 375 pt/dp，高度 ≥ 480 pt/dp。

**标注要求**：在 prd.html 每个 PAD 帧区块末尾以文字说明卡标注上述规则；spec.md 在 PAD 帧状态枚举表末尾增加一行「窗口化-手机布局触发态」，触发前提为「窗口宽度 < 440pt/dp」，备注「布局同手机帧」。

### `[Should]` PAD 交互一致性原则

PAD 帧的交互设计**以对应手机帧为基准**，不得：
- 新增手机帧中不存在的功能按钮或操作入口
- 改变手机帧中已确立的交互触发方式
- 仅因「PAD 屏幕更大」而升级或调整交互模式

**允许偏差的唯一条件**：手机端方案在 PAD 横屏宽度下存在布局结构性冲突，且须在 interaction-card 中明确注明偏差原因。

### PAD 横屏末位固定操作栏（贴底栏）

PAD 横屏（`.tablet-frame`）页面若有末位固定操作栏（"取消 / 保存"贴底按钮栏等），与桌面端一致：**`[Must]` 必须用 pub 组件 `.fb-action-bar`**（v3 统一类，详 `fb-fallback-manifest.md §3.11`；旧 `.fb-frame-bottom-bar` 已 deprecated 别名仍生效），在 `.tablet-frame` 内任意嵌套深度均可（CSS 变量就近继承自动取 sticky 贴底行为）；**`[Must]` 禁止** PM 写 inline `position:sticky` / `bottom:0` / `background` 等真源管控属性（S4-28 v3 真源匹配 + CSS 变量继承，旧 v1 inline 启发式 5/5 FP 已下线）。**手机帧（`.phone-frame`）底栏不适用本组件**（直接子层 WARN）—— 移动端按 §二 手机帧布局 + `proto_cross_platform.md` 移动端操作布局规范（navbar 顶部 + 「···」菜单方案）处理。

## 四、弹框与浮层嵌套规则

弹窗（Modal）、底部抽屉（Bottom Sheet）、Popover、Toast 等覆盖层，**必须以「叠加态」方式绘制在父级页面帧内**，不得单独作为独立帧展示：

- 父级帧呈现完整遮罩层（`rgba(0,0,0,0.5)` 半透明覆盖）+ 弹窗/抽屉内容
- 弹窗内部触点使用「弹窗前缀 D」独立编号（如 `D01`、`D02`）
- 唯一例外：底部抽屉内容极长（超过帧高度 70%）且需单独说明内部多级滚动结构时，可额外绘制「[辅助帧-抽屉内部结构]」帧，不计入状态枚举表
