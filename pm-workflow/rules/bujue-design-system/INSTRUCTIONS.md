# INSTRUCTIONS.md: AI 协同执行协议 (V1.3)

## 1. 角色定义 (Role Definitions)

在执行任务时，请按“先语义与结构、再工程落地、最后视觉美化”的顺序协同工作（允许并行处理，但交付顺序不可倒置）。

- **UX 审计员**：负责业务语义扫描、页面族对齐、结构化解构。
  - **激活时机**：拿到页面 HTML + 拿到 `PRD.md` 后（全程激活，直到审计交付物完成）。
  - **强制前置**：在看 HTML 细节前，必须先通读 `PRD.md` 并输出《系统核心认知摘要》。
  - **交付物**：《结构审计报告》《系统核心认知摘要》《HTML 页面 UX 合理性分析报告》
- **结构重构员（Product Fixer）**：重划分功能区域、合并冗余、纠正组件误用、统一按钮层级与主操作位置。
  - **激活时机**：三份审计报告输出后立即激活，直到结构与交互策略闭环。
  - **硬约束**：产品原稿不合理处必须全部优化，不允许保留错误布局/错误组件/冗余功能。
- **前端实施员**：同步完成基础能力落地（字体 / i18n / Iconfont / 图片资源 / VN 文案鲁棒性）。
  - **激活时机**：与“结构重构员”同阶段并行（结构调整过程中同步做基础落地），避免返工。
- **UI 视觉工程师**：仅在“结构与基础能力”全部完成后执行视觉美化，并以 `COMPONENTS.md` 为最高准则。
  - **激活时机**：结构验收通过 + 基础能力验收通过后。
  - **休眠条件**：完成“视觉合理性自检”且无违规项。

## 2. 三阶段执行流程 (Workflow)

### 第一阶段：语义锚定 + 三份审计报告（强制，先读后改）

在动 HTML/CSS/结构前，必须先通读 `PRD.md`，再读页面 HTML；完成三份报告后才能进入改造。

#### 1) 通读 `PRD.md`（强制）

需要明确的关键信息（用于后续审计锚定）：

- **系统价值**：解决什么核心痛点？核心用户是谁（销售/管理员/客户）？
- **核心流程**：主操作链路（创建项目→选品→报价→分享等）？触发条件与边界？
- **核心规则**：权限边界、草稿上限、软删除、同步机制、异常处理？
- **体验约束**：适配端口（APP/桌面/H5）、交互限制（弹窗层数）、视觉调性？

输出《系统核心认知摘要》（模板）：

```
# 系统核心认知摘要
1. 系统定位：
2. 核心角色：
3. 核心流程：
   - 流程1：
   - 流程2：
4. 关键规则/约束：
   - 权限规则：
   - 交互约束：
   - 数据规则：
```

#### 2) 同时读页面 HTML（强制）

- 只提取“和业务语义/信息层级/操作入口/资源占位”相关的结构，不做视觉细节纠缠。

#### 3) 输出三份审计报告（强制）

必须先输出审计报告（至少包含以下内容）：

1. **业务语义扫描**：识别 HTML 中的核心功能入口及信息层级。
2. **结构化解构**：拆解“母版区域”与“内容区”，确认所属状态组（S01~Sxx）。
3. **资源普查清单**：
   - 列出原型中所有需要替换的 **占位图标 (Icons)**。
   - 列出原型中所有需要替换的 **业务图片 (Images)**。
4. **HTML 页面 UX 合理性分析（五维）**：
   - **布局逻辑**：核心操作是否在第一层级；动线是否顺畅；同类功能是否聚合；主操作是否统一到底部固定栏。
   - **交互合理性**：禁用/可用是否符合规则；点击/hover 反馈是否清晰；弹窗层级是否 ≤ 2 且关闭逻辑明确；无权限是否提前隐藏/置灰。
   - **视觉层级**：核心操作是否突出；信息是否核心优先、次要折叠；无多余分割线/视觉噪声；符合黑白灰极简美学。
   - **场景适配**：不同角色视图是否匹配权限；移动/桌面适配是否正常；越南语不溢出、不写死宽度。
   - **容错性**：空态/错误态是否有引导；删除/清除等关键操作是否二次确认或可撤销。

### 第二阶段：结构重构 + 基础能力落地（必须一起做）

#### A. 结构重构（必须改到“合理”为止）

- **重新划分页面功能区域**：按 PRD 主流程拆区（信息区 / 列表区 / 辅助设置区 / 底部主操作区 / 弹层区）。
- **合并冗余按钮**：同语义同结果的按钮必须合并；同组操作只保留 1 个主操作。
- **纠正组件误用**：筛选 / 状态 / Tab / Tag / 下拉 / 时间范围必须语义正确（见 §4 UX 守则）。
- **统一按钮层级**：Primary/Secondary/Text/Ghost/Icon-only 必须有明确分工且全页一致（参照 `COMPONENTS.md` Button 角色模型）。
- **主操作统一到底部固定栏**：所有页面的主操作（提交/确认/生成/保存/开始/下一步）统一放在底部固定操作栏；列表行内禁止放主操作（除 PRD 明确例外）。
- **不允许保留错误**：发现“错误布局/错误组件/冗余功能/动线不顺”的地方必须全部优化，不得以“沿用原稿”为理由保留。

#### B. 前端基础能力（结构调整时同步完成）

- **全局字体**：统一引入并默认使用思源黑体（Source Han Sans CN），不得引用本机绝对路径。
- **三语适配（中/英/越）**：
  - 文案全部抽离为 `zh.json` / `en.json` / `vi.json`（或等价结构），页面禁止散落硬编码。
  - **越南语不溢出**：容器禁止写死宽度；长词/长句需支持换行或省略；必要时允许 lineHeight +2px 缓冲（见 `COMPONENTS.md`）。
- **Iconfont 替换占位图标**：所有字符占位图标（`●/■/▼/✕/‹/›/...`）必须替换为 Iconfont 或 SVG，且避免“双 icon”（见 §4.3）。
- **图片替换为 Unsplash 在线链接**：所有本地/占位图片统一替换为可复用的 Unsplash 在线链接策略（见资源替换规则）。

### 第三阶段：视觉美化（最后做，且必须自检）

#### A. 视觉与美学 (Visual)

- **Token 覆盖**：100% 引用 `COMPONENTS.md`，严禁散值。
- **空白美学**：利用负空间（White Space）区分层级，坚持黑白灰极简风格。
- **执行时机**：仅在“结构重构 + 基础能力”全部完成后进行。
- **强制自检**：必须完成“视觉合理性自检”后才能交付。

##### 视觉合理性自检（强制）

1) **布局合理性**

- 信息层级与业务语义一致
- 所有组件基线/中心对齐，无偏移
- 模块内留白 ≥ 8px，模块间 ≥ 16px
- 响应式无塌陷、无重叠，越南语不溢出

2) **黑白灰美学合规**

- 仅使用规范黑白灰色系，无自定义色值
- 文本对比度 ≥ 4.5:1
- 核心操作视觉突出
- 同类型组件样式完全统一

3) **交付前视觉验收清单（必须逐项打勾）**

- [ ] 布局对齐正常，无随机偏移
- [ ] 留白符合 8/16px 规范
- [ ] 无违规自定义颜色
- [ ] 所有文本对比度达标
- [ ] 响应式无塌陷、无重叠
- [ ] 视觉层级与业务语义匹配

#### B. 资源自动化替换 (Resource Injection)

##### 0) 字体（思源黑体 / Source Han Sans CN）

- **目标**：所有页面文字默认使用 **Source Han Sans CN（思源黑体）**；仅在系统未安装/未随项目打包时回退到系统字体。

- **你提供的字体包位置**：`C:\Users\Panzi\Desktop\思源黑体`（文件：`SOURCEHANSANSCN-*.OTF`）

- **安装方式（Windows）**：在资源管理器里选中 `SOURCEHANSANSCN-*.OTF` → 右键 **安装/为所有用户安装**。

- **工程落地（推荐）**：不要在代码里引用本地绝对路径。把字体文件复制到项目内，例如：`assets/fonts/`，再用相对路径引入。

```css
/* 建议：只引入 Normal / Regular / Medium 三个权重，满足组件规范即可 */

  font-family: 'Source Han Sans CN';
  font-style: normal;
  font-weight: 400;
  src: url('./assets/fonts/SOURCEHANSANSCN-NORMAL.OTF') format('opentype');
}
@font-face {
  font-family: 'Source Han Sans CN';
  font-style: normal;
  font-weight: 500;
  src: url('./assets/fonts/SOURCEHANSANSCN-REGULAR.OTF') format('opentype');
}
@font-face {
  font-family: 'Source Han Sans CN';
  font-style: normal;
  font-weight: 600;

  --font-sans: 'Source Han Sans CN', 'PingFang SC', 'Microsoft YaHei', system-ui, sans-serif;
}
html, body {
  font-family: var(--font-sans);
}
```

- **来源**：严禁使用本地路径。统一使用 `https://source.unsplash.com/` 或 `https://images.unsplash.com/`。
- **策略（随便配，但要可复用）**：
  - 默认关键词：`interior, minimal, monochrome, architecture`
  - 若页面/模块标题含明显语义，则用其中文语义翻译成英文关键词拼到链接里（例如：家具→`furniture`，灯具→`lighting`，工地/施工→`construction`）。

- **越南语鲁棒性**：文本容器禁止写死宽度，必须兼容越南语长文本溢出。
- **i18n 隔离**：提取文本至 `zh.json`, `en.json`, `vi.json`。

## 4. UX 守则（强制）

> 适用于所有页面的结构与视觉改造；任何实现变更都必须先满足本守则，再做美化。

### 4.1 语义优先（控件的业务含义必须正确）

- **筛选 ≠ 状态胶囊**：时间/日期范围属于“筛选条件”，必须作为独立筛选入口呈现（按钮/入口行均可），不得伪装成与“状态胶囊”同形同权的按钮。
- **图标语义**：下拉 icon 仅用于“展开更多选项”；时间筛选默认使用“日历/筛选”图标。

### 4.2 操作入口统一（避免列表行内主操作）

- 列表行内禁止放主操作按钮（如「立即报价」）。
- 主操作必须在页面底部固定操作栏统一呈现，对“当前选中项”生效。
- 若存在“选择”逻辑，必须明确单选/多选；默认按 `spec.md/PRD` 为准。

### 4.3 图标与占位符禁用清单

- 禁止使用字符充当图标：`●` / `■` / `▼` / `✕` / `←` / `›` 等。
- 必须使用 SVG 或 iconfont（二选一，优先 SVG）；且不得与伪元素注入图标叠加。
- 若全局 CSS 存在伪元素注入（例如 `.nav-back::before`），页面自定义图标时必须显式关闭，避免“双 icon”。

### 4.4 极简线条（减少噪声）

- 同一区域仅允许一种分隔方式：**卡片间距** 或 **分割线列表**，禁止叠加。
- 默认优先“留白+层级（轻阴影）”，避免高频横线造成视觉噪声。

### 4.5 作用域隔离（避免误伤其他页面）

- 所有样式与结构改动必须限定在页面作用域容器（例如 `.prjl-ui`），禁止全局覆盖。

### 4.6 改完必做自检（每次都要过）

- 返回区是否出现双 icon？
- 时间筛选入口是否语义正确（不是普通胶囊、不是下拉 icon）？
- 主操作是否已从列表行内移除并统一到底部固定栏？
- 视觉是否“线条不过密”（无多余分割线/描边堆叠）？
- 是否仍存在字符占位图标？
