## 三、桌面端组件规范

### 分页

桌面端优先使用分页控件（不使用移动端的上滑加载更多）：
- 分页控件置于列表底部右侧
- 含上/下页、页码、每页条数选择器

### Hover 态记录规则

若某元素有 hover 态，在触点卡片中新增一条，触发方式填「鼠标悬浮」，描述样式变化（背景色、Tooltip 出现等）；仅样式变化、无业务逻辑的可合并为末尾注记行。

### 键盘交互

桌面端须在触点卡片中标注键盘入口：

| 触发方式 | 典型场景 |
|---------|---------|
| 键盘 Tab | 表单字段间焦点切换 |
| 键盘 Enter / Space | 确认按钮、选择项 |

### 末位固定操作栏（贴底栏）

桌面端页面若有末位固定操作栏（典型："取消 / 保存"贴底按钮栏、表单提交栏），**`[Must]` 必须用 pub 组件 `.fb-action-bar`**（v3 统一类，详 `fb-fallback-manifest.md §3.11`；旧 `.fb-frame-bottom-bar` 已 deprecated 别名仍生效）：

- `.fb-action-bar` 在 `.desktop-frame` 内（任意嵌套深度）自动取 sticky 贴底行为（CSS 变量就近继承）
- sticky 贴底由该 class 真源 CSS 变量管控 — **`[Must]` 禁止** PM 在底栏元素上写 inline `style="position:sticky"` / `bottom:0` / `background:...` 等真源管控属性（旧 v1 inline 启发式 5/5 FP 已下线，S4-28 v3 真源 class 匹配 + CSS 变量继承）
- 按钮用 `.fb-btn` 系列，主操作 `fb-btn-primary` 居右

## 五、弹框与浮层

桌面端 Toast 位置：屏幕**左下角**，距左 24px、距底 24px（区别于移动端的底部居中）。

弹窗（Modal）、抽屉等浮层规则与 APP 端一致，须以「叠加态」方式绘制在父级帧内，不得单独作为独立帧展示。
